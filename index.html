Technical Deep-Dive — Mark
Why Google Sheets + Statistical Models Don't Mix

Statistical Models (Deterministic)

  • Require clean, typed data — numbers stored as text cause errors
  • Client names differing across sheets break joins
  • Outdated sheets with ghost clients pollute datasets
  • Strict schema enforcement needed — Sheets cannot provide this

LLMs (Semantic)

  • Can ingest unstructured data — but feeding all our data is prohibitively expensive
  • Inconsistent data causes hallucinations
  • Outputs sound accurate but are polluted by data quality issues
  • Short-term insights possible — long-term historical analytics are not
Foundation Principle
You cannot build a predictive model without knowing what data exists, where it lives, and whether it is clean. Organization must come first.

Data Integrity — Mark
Schema Enforcement & WorkStack OS

Google Sheets Limitations

  • Schema can be changed on a whim — column names, types, layouts
  • No data integrity enforcement — relies entirely on human discipline
  • Formulas break when sheet layouts change
  • Manual client name duplication across sheets — inconsistency inevitable

Nexus v2 & WorkStack OS

  • Strict schema — impossible to store unclean data
  • Enforced by forms, not human memory
  • Relationships maintained by foreign keys — no manual duplication
  • WorkStack OS: EOD reporting with structured data entry by design
WorkStack OS — Scheduled for May, In Progress
Currently using End-of-Day reporting in WorkStack. Forms dictate the schema — migrating more operations here aligns with the data organization goal.

Every new platform acquired adds ongoing connection maintenance. My time is split between maintaining existing workflows, fixing broken automations, managing credentials, and high-value initiatives (data organization, analytics, Nexus v2).
